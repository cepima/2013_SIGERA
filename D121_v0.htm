WP 1  Mejoras en tecnologías y herramientas de modelización

 

D1.2.1  Arquitectura de un sistema modular de identificación y diagnosis

 

Introducción al problema

Hasta la actualidad se han desarrollado infinidad de técnicas de tratamiento de datos, filtración, representación de la información y sus características (feature extraction, feature extensión, clustering), clasificación (multi-class, multi-label). Sin embargo, no existen referencias a sistemas que ensamblen de manera automática las mejores técnicas disponibles para construir un sistema de diagnosis de procesos, que tan solo se han explorado muy recientemente.

Arquitectura del sistema

Es por ello que se desarrollaron un conjunto de programas informáticos, también denominados módulos, cuyo objetivo es la comunicación entre aplicaciones y/o plataformas, de manera que se permite dicha comunicación de manera on-line u off-line. La arquitectura del sistema sería el siguiente, donde se denomina el concepto y la tecnología utilizada entre paréntesis.

Fig. 1. Esquema de arquitectura modular.

El módulo principal es el que se denomina Thot, que es un RSS, es decir, un formato XML para compartir contenido. Concretamente, Thot permite compartir archivos en formato XML generados por los distintos módulos que forman parte de la estructura, de modo que cualquier bloque puede adquirir o escribir en el archivo que le sea necesario.

Estos distintos módulos corresponden a los diferentes elementos que pueden intervenir en una planta real o una simulación: el proceso, ya sea real o simulado, una interfaz gráfica, una ontología y una optimización.

El proceso corresponde a la realidad que se está tratando y analizando. Dada la complejidad de la estructura, es necesario que el proceso sea capaz de establecer una comunicación que el RSS pueda entender y viceversa. Es por ello que se requiere de una plataforma que permita transferir la información del proceso, por ejemplo, Matlab, a Thot, denominado T.G.C.

Fig. 2. Interfaz de TGC.

 

La interfaz gráfica de usuario, o HMI del inglés Human Machine Interface, es una herramienta para el usuario que le permite, por un lado, ver la información más relevante del proceso y, por otro lado, gestionar el proceso, refiriéndose a esto como: inicializar el proceso (o simulación), optimizar el proceso o enviar órdenes, como cambiar la producción de una planta, qué cantidad de qué producto se envía a un mercado, etc.

Fig. 3. Interfaz gráfica de usuario.

La ontología, aplicada mediante la herramienta OntoCEP, es una catalogación de los conceptos del problema, así como la posible relación entre estos e información detallada, como valor, fecha de ese valor, etc. Además, también permite gestionar el uso de GAMS, así como los resultados, que interesarán tanto para optimizaciones de procesos como para la diagnosis.

Fig. 4. Interfaz de OntoCEP.

El principal caso de estudio para el desarrollo e implementación será el proceso Tennessee-Eastman, debido a que es un proceso muy estudiado y analizado que permite trabajar tanto con Matlab, que realiza la simulación, como el diagnóstico de los datos mediante GAMS (Askarian, 2016), (Monroy, 2012).
